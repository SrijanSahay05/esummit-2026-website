# Lessons

This directory is a **preference ledger** — a record of the user's architectural decisions, code style preferences, and other conventions that should be respected throughout the project.

## Purpose

When working on this project, agents and collaborators should consult these files to understand:
- **How** the user wants code structured and written
- **Why** certain architectural decisions were made
- **What** conventions to follow

## Files

| File | Contents |
|------|----------|
| `architecture.md` | Architecture decisions, framework choices, component patterns |
| `development-style.md` | Code style, formatting, naming conventions, tooling preferences |
| `misc.md` | Miscellaneous preferences that don't fit elsewhere |

## How to Use

- Read these files before starting any new feature or significant change
- Update them when the user expresses a new preference or changes an existing one
- These are living documents — they evolve as the project matures
